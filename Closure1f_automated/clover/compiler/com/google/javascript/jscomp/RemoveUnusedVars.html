34    /**
35    * Garbage collection for variable and function definitions. Basically performs
36    * a mark-and-sweep type algorithm over the JavaScript parse tree.
37    *
38    * For each scope:
39    * (1) Scan the variable/function declarations at that scope.
40    * (2) Traverse the scope for references, marking all referenced variables.
41    * Unlike other compiler passes, this is a pre-order traversal, not a
42    * post-order traversal.
43    * (3) If the traversal encounters an assign without other side-effects,
44    * create a continuation. Continue the continuation iff the assigned
45    * variable is referenced.
46    * (4) When the traversal completes, remove all unreferenced variables.
47    *
48    * If it makes it easier, you can think of the continuations of the traversal
49    * as a reference graph. Each continuation represents a set of edges, where the
50    * source node is a known variable, and the destination nodes are lazily
51    * evaluated when the continuation is executed.
52    *
53    * This algorithm is similar to the algorithm used by {@code SmartNameRemoval}.
54    * {@code SmartNameRemoval} maintains an explicit graph of dependencies
55    * between global symbols. However, {@code SmartNameRemoval} cannot handle
56    * non-trivial edges in the reference graph ("A is referenced iff both B and C
57    * are referenced"), or local variables. {@code SmartNameRemoval} is also
58    * substantially more complicated because it tries to handle namespaces
59    * (which is largely unnecessary in the presence of {@code CollapseProperties}.
60    *
61    * This pass also uses a more complex analysis of assignments, where
62    * an assignment to a variable or a property of that variable does not
63    * necessarily count as a reference to that variable, unless we can prove
64    * that it modifies external state. This is similar to
65    * {@code FlowSensitiveInlineVariables}, except that it works for variables
66    * used across scopes.
67    *
68    */
