32    /**
33    * Compute the "liveness" of all local variables. A variable is "live" at a
34    * point of a program if the value it is currently holding might be read later.
35    * Otherwise, the variable is considered "dead" if we know for sure that it will
36    * no longer be read. Dead variables are candidates for dead assignment
37    * elimination and variable name sharing. The worst case safe assumption is to
38    * assume that all variables are live. In that case, we will have no opportunity
39    * for optimizations. This is especially the case within a TRY block when an
40    * assignment is not guaranteed to take place. We bail out by assuming that
41    * all variables are live.
42    * <p>
43    * Due to the possibility of inner functions and closures, certain "local"
44    * variables can escape the function. These variables will be considered as
45    * global and they can be retrieved with {@link #getEscapedLocals()}.
46    *
47    */
