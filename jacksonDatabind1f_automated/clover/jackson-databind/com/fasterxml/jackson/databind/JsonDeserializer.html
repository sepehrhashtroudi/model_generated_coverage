11    /**
12    * Abstract class that defines API used by {@link ObjectMapper} (and
13    * other chained {@link JsonDeserializer}s too) to deserialize Objects of
14    * arbitrary types from JSON, using provided {@link JsonParser}.
15    *<p>
16    * Custom deserializers should usually not directly extend this class,
17    * but instead extend {@link com.fasterxml.jackson.databind.deser.std.StdDeserializer}
18    * (or its subtypes like {@link com.fasterxml.jackson.databind.deser.std.StdScalarDeserializer}).
19    *<p>
20    * If deserializer is an aggregate one -- meaning it delegates handling of some
21    * of its contents by using other deserializer(s) -- it typically also needs
22    * to implement {@link com.fasterxml.jackson.databind.deser.ResolvableDeserializer},
23    * which can locate dependant deserializers. This is important to allow dynamic
24    * overrides of deserializers; separate call interface is needed to separate
25    * resolution of dependant deserializers (which may have cyclic link back
26    * to deserializer itself, directly or indirectly).
27    *<p>
28    * In addition, to support per-property annotations (to configure aspects
29    * of deserialization on per-property basis), deserializers may want
30    * to implement
31    * {@link com.fasterxml.jackson.databind.deser.ContextualDeserializer},
32    * which allows specialization of deserializers: call to
33    * {@link com.fasterxml.jackson.databind.deser.ContextualDeserializer#createContextual}
34    * is passed information on property, and can create a newly configured
35    * deserializer for handling that particular property.
36    *<p>
37    * If both
38    * {@link com.fasterxml.jackson.databind.deser.ResolvableDeserializer} and
39    * {@link com.fasterxml.jackson.databind.deser.ContextualDeserializer}
40    * are implemented, resolution of deserializers occurs before
41    * contextualization.
42    */
