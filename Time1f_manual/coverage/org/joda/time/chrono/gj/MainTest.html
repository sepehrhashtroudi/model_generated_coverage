32    /**
33    * Tests either the Julian or Gregorian chronology from org.joda.time.chrono.gj
34    * against the implementations in this package. It tests all the date fields
35    * against their principal methods.
36    * <p>
37    * Randomly generated values are fed into the DateTimeField methods and the
38    * results are compared between the two chronologies. If any result doesn't
39    * match, an error report is generated and the program exits. Each time this
40    * test program is run, the pseudo random number generator is seeded with the
41    * same value. This ensures consistent results between test runs.
42    * <p>
43    * The main method accepts three optional arguments: iterations, mode, seed. By
44    * default, iterations is set to 1,000,000. The test will take several minutes
45    * to run, depending on the computer's performance. Every 5 seconds a progress
46    * message is printed.
47    * <p>
48    * The mode can be either 'g' for proleptic gregorian (the default) or 'j' for
49    * proleptic julian. To override the default random number generator seed, pass
50    * in a third argument which accepts a long signed integer.
51    *
52    * @author Brian S O'Neill
53    */
