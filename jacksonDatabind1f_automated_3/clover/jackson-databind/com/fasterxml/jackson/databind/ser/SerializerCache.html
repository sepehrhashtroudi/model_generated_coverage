8    /**
9    * Simple cache object that allows for doing 2-level lookups: first level is
10    * by "local" read-only lookup Map (used without locking)
11    * and second backup level is by a shared modifiable HashMap.
12    * The idea is that after a while, most serializers are found from the
13    * local Map (to optimize performance, reduce lock contention),
14    * but that during buildup we can use a shared map to reduce both
15    * number of distinct read-only maps constructed, and number of
16    * serializers constructed.
17    *<p>
18    * Since version 1.5 cache will actually contain three kinds of entries,
19    * based on combination of class pair key. First class in key is for the
20    * type to serialize, and second one is type used for determining how
21    * to resolve value type. One (but not both) of entries can be null.
22    */
